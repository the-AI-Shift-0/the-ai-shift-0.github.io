Demo 2 Interactive Voice Demo (Local Server)

Why you need to do the setup

The voice recording and AI prompt generation in this demo require a local server running on your laptop. The live website on GitHub Pages can't do this on its own — it needs a small Python server running in the background on your machine to handle the AI calls. Setup takes about 5 minutes and only needs to happen once.

Part 1 One-Time Server Setup
Step 1

Create a new empty folder

Create a new empty folder anywhere on your computer — your Desktop is fine. Name it something like ai-shift-demo. You can right-click your Desktop and choose New Folder.

Part 2 Running the Demo
Before you start

Make sure the server is running

Check your terminal — you should see something like Running on http://localhost:5000. Then open the demo:

Open Prompt Builder Demo
Demo walkthrough

Step by step on screen

  • Main page: Show the input area. Click the microphone button and speak your task (10-15 seconds is enough). Example: "Write a follow-up email to a client who hasn't replied in two weeks."
  • Let it transcribe: The voice gets converted to text and appears in the input box automatically.
  • Click "Build my prompt": The AI analyses your task and builds an optimised prompt using prompting best practices.
  • Results page: Show the generated prompt. Point out how it's structured — role, context, task, and output format are all spelled out clearly.
  • The payoff: Paste the generated prompt into Claude or ChatGPT. Show the output. Then show what you get from a basic one-liner. The contrast makes the point.
Suggested opening line
"Most people type a one-liner into ChatGPT and wonder why the results are average. This tool takes what you actually want - in plain English - and turns it into a proper prompt that gets you dramatically better output."
Filming tips

Things to keep in mind

Use a real business task

Sales follow-up, a refund policy, a job listing — anything an SMB owner would actually do.

Show the before/after

Run the same task as a basic prompt in ChatGPT first, then show the output using the generated one. The difference sells it.

Keep voice input short

10-15 seconds of natural speech is enough. No need for a perfect sentence.

Server must be running

If the voice button doesn't work, check your terminal. The server must be running at localhost:5000 for the demo to work.
