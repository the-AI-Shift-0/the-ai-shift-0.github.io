Demo Visual Guide - AI CEO

Production spec for Demo 1 (simulation) and Demo 2 (live Claude). Last updated: 2026-05-12.

How to read this doc: Everything here is directional - the concept, structure, and intent. Wireframe values, question chip wording, metric labels, and panel names are illustrative placeholders. AP (with Claude Code) builds the simulation. AP fills the actual content. Sai navigates it on camera. Nothing in here is a pixel-for-pixel requirement.
Decisions From The Call
Demo 1 = Simulation. Claude Code builds a set of linked HTML pages that mimic the branded dashboard. Sai navigates through it live while recording - no real backend needed.
Demo 2 = Live Claude Web Interface. Real Claude Project, real prompts. Recorded on Sai's Claude account - Sai uses the web interface regularly and already has context on what it can do.
Blurring updated (May 12 call). During dev/review - all blurs off so AP can review numbers. For recording: blur revenue/financial dollar amounts only. Percentage gaps, incident counts, hours - always show. Names: blur only when shown in negative context (e.g. causing operational drag). Positive name mentions - show.
Recording format: Sai records himself going through the demo naturally with a floating PiP head (Loom-style circle). Voiceover is live, not post-recorded. Minimal editing needed if the simulation is well-built.
Demo 1 direction updated - May 2026. The 7-step sequential flow has been replaced with a 3-level dashboard-first structure. Reason: multi-step forces viewers to extrapolate relationships across screens ("how does step 5 relate to step 2?") - cognitive overhead that kills impact. New rule: show everything holistically at Level 2 first, then allow one drill-down. Max 3 levels, always. See full spec below.
Demo 1 - Simulation Architecture

Four screens, three levels of interaction. Level 2 (the holistic dashboard) is the centrepiece - everything that matters is visible simultaneously. No sequential steps. No extrapolating. Sai clicks a pre-filled question chip, the full dashboard appears, then he drills one level deeper on one area.

Demo 2 - Live Claude Web Interface
This is the real thing. No simulation. Viewer thinks "I can do this right now." Every step is something they already have access to.
1
Pre-recording: Use Sai's Claude account
Sai uses the Claude web interface regularly - this is his natural environment. Set up the AI Co-CEO Claude Project on Sai's account with the MD context file, calendar, and Slack connected. Sai tests the Sunday assessment prompt end-to-end before going on camera. Apurv uses Claude Code primarily and does not set this up.
2
Show the Claude Project interface
Open the project. Show the sidebar - project name, integrations connected (calendar, Slack icons visible). Say: "This is a Claude Project - different from a regular chat. It holds persistent context so the AI remembers your business every time."
3
Show the MD context file
Open the project knowledge file. Scroll through it - viewer sees it's detailed (not just "I'm a CEO"). Say: "This is the context file - it tells the AI who you are, your targets, your team, your revenue lines. We've pre-built this for you."
4
Run the Sunday assessment live
Type the same assessment prompt as Demo 1. Let the response stream in - don't cut. The AI pulls calendar and Slack. Show the output: same kind of cascade + diagnosis as Demo 1 but in plain interface. Don't skip any of the generation - the streaming IS the proof it's working.
5
Show the week plan being generated
Ask it to plan the week. Show calendar entries being created with context. Viewer sees: specific priorities, why each one matters, linked to targets. Key message: "The viewer is watching and thinking - I can do this right now."
Apurv's knowledge gap (not Sai's): Apurv primarily uses Claude Code, not the web interface. Demo 2 setup and testing is Sai's domain - he knows the web interface. Before recording, Sai runs the full assessment flow once on his account to confirm calendar + Slack pull correctly. Only record once it works clean end-to-end.

Recording Setup

Build simulation in Claude Code first. Test all 4 screens (L0–L3) flow correctly before shooting.

Open simulation in browser, fullscreen (1920×1080). No browser chrome visible if possible.
3
Screen record + face cam simultaneously. Floating PiP head (circle, Loom-style), bottom-right corner.
4
Sai speaks live while navigating - don't post-record VO. Natural pauses and emphasis matter.
5
AP sends Sai a recording pathway - which question chip to click on L1, which panel to drill on L2. Follow it exactly. No improvising the drill path.
Studio note: Recording in studio with teleprompter may be awkward for screen interaction. Consider whether Demo 1 is recorded at home (screen share) vs studio (teleprompter). Decide before shoot day.

1
Use Sai's Claude account. Sai sets up the AI Co-CEO Project on his own account - calendar + Slack connected, MD context file loaded.
